# 执行计划 (Executing Plans)

## 概述

加载计划，进行批判性评估，执行所有任务，并在完成后进行汇报。

**开始时请宣告：** “我正在使用 `executing-plans` Skill 来执行此计划。”

**注意：** 请告知你的人类伙伴，如果能够访问子 Agent (subagents)，本系列 Skill (Superpowers) 的表现会出色得多。如果在支持子 Agent 的平台（如 Claude Code 或 Codex）上运行，其工作质量将显著提高。如果子 Agent 可用，请使用 `superpowers:subagent-driven-development` 代替此 Skill。

## 流程

### 第一步：加载与评估计划
1. 阅读计划文件。
2. 进行批判性评估 —— 识别出任何关于该计划的疑问或担忧。
3. 如果有担忧：在开始前与你的人类伙伴沟通。
4. 如果没有担忧：创建 `TodoWrite` 任务列表并继续。

### 第二步：执行任务

对于每一项任务：
1. 将其标记为 `in_progress`（进行中）。
2. 严格遵循每一个步骤（计划中已拆分为易于处理的小步骤）。
3. 按照指定的步骤运行验证。
4. 将其标记为 `completed`（已完成）。

### 第三步：完成开发

在所有任务完成并经过验证后：
- 宣告：“我正在使用 `finishing-a-development-branch` Skill 来完成此项工作。”
- **必须调用的子 Skill：** 使用 `superpowers:finishing-a-development-branch`。
- 遵循该 Skill 的指导来验证测试、展示选项并执行最终选择。

## 何时停止并寻求帮助

**在以下情况下立即停止执行：**
- 遇到阻塞（缺少依赖项、测试失败、指令不明确）。
- 计划存在关键性缺口导致无法开始。
- 你不理解某条指令。
- 验证反复失败。

**请针对不明确的地方进行询问，而不是瞎猜。**

## 何时返回之前的步骤

**在以下情况返回“评估计划”（第一步）：**
- 伙伴根据你的反馈更新了计划。
- 需要重新考虑基本的实现方案。

**不要强行冲破阻塞** —— 停下来询问。

## 谨记
- 首先对计划进行批判性评估。
- 精确遵循计划步骤。
- 不要跳过验证。
- 当计划提到某些 Skill 时，请查阅对应 Skill。
- 遇阻即停，不要瞎猜。
- 未经用户明确同意，绝不在 `main`/`master` 分支上开始实施。

## 集成

**必需的工作流 Skill：**
- **`writing-plans`** —— 创建此 Skill 执行的计划。
