# 完成开发分支 (Finishing a Development Branch)

## 概述

通过展示清晰的选项并处理所选的工作流，指导开发工作的收尾。

**核心原则：** 验证测试 -> 展示选项 -> 执行选择 -> 清理环境。

**开始时请宣告：** “我正在使用 `finishing-a-development-branch` Skill 来完成此项工作。”

## 流程

### 第一步：验证测试

**在展示选项之前，必须验证测试已通过：**

```bash
# 运行项目的测试套件
npm test / cargo test / pytest / go test ./...
```

**如果测试失败：**
```
测试失败（共 <N> 项失败）。在完成前必须修复：

[展示失败详情]

在该测试通过前，无法进行合并 (Merge) 或创建 PR。
```

此时应停止执行，不要进入第二步。

**如果测试通过：** 进入第二步。

### 第二步：确定基准分支 (Base Branch)

```bash
# 尝试常见的基准分支
git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null
```

或者询问：“该分支是从 `main` 分出的 —— 是否正确？”

### 第三步：展示选项

准确展示以下 4 个选项：

```
实现已完成。你希望接下来做什么？

1. 本地合并回 <基准分支>
2. 推送 (Push) 并创建 Pull Request (PR)
3. 保持分支现状（我稍后会处理）
4. 丢弃此工作
5. 请选择一个选项。
```

**不要添加额外解释** —— 保持选项简洁。

### 第四步：执行选择

#### 选项 1：本地合并

```bash
# 切换到基准分支
git checkout <基准分支>

# 拉取最新代码
git pull

# 合并功能分支
git merge <功能分支>

# 验证合并后的测试结果
<测试命令>

# 如果测试通过
git branch -d <功能分支>
```

然后执行：清理工作树（第五步）。

#### 选项 2：推送并创建 PR

```bash
# 推送分支
git push -u origin <功能分支>

# 创建 PR
gh pr create --title "<标题>" --body "$(cat <<'EOF'
## 概述
<变更内容简述，2-3 个要点>

## 测试计划
- [ ] <验证步骤>
EOF
)"
```

然后执行：清理工作树（第五步）。

#### 选项 3：保持现状

汇报：“保留分支 <名称>。工作树保留在 <路径>。”

**不要清理工作树。**

#### 选项 4：丢弃

**首先进行确认：**
```
这将永久删除：
- 分支 <名称>
- 所有提交：<提交列表>
- 位于 <路径> 的工作树

请输入 'discard' 以确认。
```

等待用户输入精确的确认词。

如果确认：
```bash
git checkout <基准分支>
git branch -D <功能分支>
```

然后执行：清理工作树（第五步）。

### 第五步：清理工作树

**对于选项 1、2、4：**

检查是否处于工作树中：
```bash
git worktree list | grep $(git branch --show-current)
```

如果是：
```bash
git worktree remove <工作树路径>
```

**对于选项 3：** 保留工作树。

## 快速参考

| 选项 | 合并 | 推送 | 保留工作树 | 清理分支 |
|--------|-------|------|---------------|----------------|
| 1. 本地合并 | ✓ | - | - | ✓ |
| 2. 创建 PR | - | ✓ | ✓ | - |
| 3. 保持现状 | - | - | ✓ | - |
| 4. 丢弃 | - | - | - | ✓ (强制) |

## 常见错误

### 跳过测试验证

- **问题：** 合并了损坏的代码，或创建了失败的 PR。
- **修复：** 在提供选项前始终先验证测试。

### 开放式提问

- **问题：** “我接下来该做什么？” -> 产生歧义。
- **修复：** 准确展示上述 4 个结构化选项。

### 自动清理工作树

- **问题：** 在可能仍需要工作树的情况下（选项 2、3）将其删除了。
- **修复：** 仅针对选项 1 和 4 进行清理。

### 丢弃工作前未确认

- **问题：** 意外删除了劳动成果。
- **修复：** 要求输入 'discard' 进行明确确认。

## 红灯信号

**绝不要：**
- 在测试失败的情况下继续。
- 在未验证合并结果测试的情况下进行合并。
- 在未确认的情况下删除工作。
- 在未明确要求的情况下进行强制推送 (Force-push)。

**始终做到：**
- 在提供选项前验证测试。
- 准确展示 4 个选项。
- 为选项 4 获取手打的确认。
- 仅针对选项 1 和 4 清理工作树。
