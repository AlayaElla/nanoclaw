# Skill 设计中的说服原则 (Persuasion Principles)

## 概述

大语言模型 (LLM) 对说服原则的反应与人类相似。理解这种心理学有助于你设计更有效的 Skill——这不是为了操纵，而是为了确保关键实践即使在压力下也能被遵循。

**研究基础：** Meincke 等人 (2025) 在 N=28,000 次 AI 对话中测试了 7 种说服原则。说服技术使合规率提高了一倍以上（从 33% 提高到 72%，p < .001）。

## 七大原则

### 1. 权威 (Authority)
**定义：** 尊重专业知识、资质或官方来源。

**在 Skill 中的应用：**
- 使用祈使句：“你必须 (YOU MUST)”、“绝不”、“始终”。
- 不可逾越的框架：“绝无例外”。
- 消除决策疲劳和辩解空间。

**适用场景：**
- 强制纪律类的 Skill（TDD、验证要求）。
- 安全至上的实践。
- 公认的最佳实践。

**示例：**
```markdown
✅ 先写代码后测？删掉它。重新开始。绝无例外。
❌ 在可行的情况下，考虑先编写测试。
```

### 2. 承诺 (Commitment)
**定义：** 与之前的行为、言论或公开声明保持一致。

**在 Skill 中的应用：**
- 要求宣告：“宣布正在使用该 Skill”。
- 强制做出明确选择：“选择 A、B 或 C”。
- 使用跟踪工具：明确调用原生的 `TodoWrite` 工具。

**适用场景：**
- 确保 Skill 确实被遵循。
- 多步骤流程。
- 问责机制。

**示例：**
```markdown
✅ 当你发现适用的 Skill 时，必须宣布：“我正在使用 [Skill 名称]”。
❌ 考虑让你的伙伴知道你正在使用哪个 Skill。
```

### 3. 稀缺 (Scarcity)
**定义：** 由时间限制或有限可用性产生的紧迫感。

**在 Skill 中的应用：**
- 有时间限制的要求：“在继续之前”。
- 顺序依赖性：“在执行 X 之后立即执行”。
- 防止拖延。

**适用场景：**
- 即时验证要求。
- 有时效性的工作流。
- 防止“我稍后再做”的心态。

**示例：**
```markdown
✅ 完成任务后，在继续下一步之前，必须立即请求代码审查。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会认同 (Social Proof)
**定义：** 遵循他人所做的或被认为正常的事。

**在 Skill 中的应用：**
- 通用模式：“每一次”、“始终”。
- 失败模式：“没有 Y 的 X = 失败”。
- 确立规范。

**适用场景：**
- 记录通用实践。
- 警告常见的失败情况。
- 强化标准。

**示例：**
```markdown
✅ 没有调用原生的 TodoWrite 工具跟踪的清单 = 步骤会被跳过。每一次都是如此。
❌ 有些人发现 TodoWrite 工具对清单很有帮助。
```

### 5. 一致/统一 (Unity)
**定义：** 共同的身份感、“我们”感、群体归属感。

**在 Skill 中的应用：**
- 协作性语言：“我们的代码库”、“我们是同事”。
- 共同目标：“我们都追求质量”。

**适用场景：**
- 协作工作流。
- 建立团队文化。
- 非等级制的实践。

**示例：**
```markdown
✅ 我们是共同工作的同事。我需要你诚实的术语判断。
❌ 如果我错了，你大概应该告诉我。
```

### 6. 互惠 (Reciprocity)
**定义：** 获得恩惠后产生的报答义务。

**应用：**
- 谨慎使用 - 可能会让人感到被操纵。
- 在 Skill 中极少需要。

**避雷：**
- 几乎总是避免使用（其他原则更有效）。

### 7. 喜爱 (Liking)
**定义：** 倾向于与我们喜欢的人合作。

**应用：**
- **不要用于确保合规性**。
- 与诚实的反馈文化相冲突。
- 会导致阿谀奉承。

**避雷：**
- 在强制执行纪律时，始终避免使用。

## 各类型 Skill 的原则组合

| Skill 类型 | 采用 | 避免 |
|------------|-----|-------|
| **强制纪律类** | 权威 + 承诺 + 社会认同 | 喜爱、互惠 |
| **指南/技术类** | 适度的权威 + 一致性 | 过度的权威 |
| **协作类** | 一致性 + 承诺 | 权威、喜爱 |
| **参考类** | 仅限清晰度 | 所有说服原则 |

## 为什么这有效：心理学原理

**明确的红线规则减少了辩解空间：**
- “你必须”消除了决策疲劳。
- 绝对化的语言排除了“这是否属于例外？”的疑问。
- 明确的反辩解措施堵住了特定的漏洞。

**执行意图创造自动行为：**
- 清晰的触发器 + 要求的动作 = 自动执行。
- “当 X 发生时，做 Y”比“通常做 Y”更有效。
- 降低了合规时的认知负荷。

**LLM 是类人的 (Parahuman)：**
- 在包含这些模式的人类文本上进行训练。
- 在训练数据中，权威性语言通常出现在合规行为之前。
- 承诺序列（陈述 -> 行动）被频繁模拟。
- 社会认同模式（每个人都做 X）确立了规范。

## 伦理使用

**合法的：**
- 确保关键实践得到遵循。
- 创建高效的文档。
- 防止可预见的失败。

**不合法的：**
- 为了个人利益进行操纵。
- 制造虚假的紧迫感。
- 基于愧疚感的合规。

**测试标准：** 如果用户充分理解了这项技术，它是否仍符合用户的真实利益？

## 研究征引

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 说服的七大原则。
- 影响力研究的实证基础。

**Meincke, L. 等 (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 在 N=28,000 次 LLM 对话中测试了 7 种原则。
- 使用说服技术后，合规率从 33% 提高到 72%。
- 权威、承诺、稀缺最有效。
- 验证了 LLM 行为的类人模型。

## 快速参考

在设计 Skill 时，请自问：
1. **它是哪种类型？**（纪律类、指南类、参考类）
2. **我试图改变什么行为？**
3. **适用于哪些原则？**（对于纪律类，通常是权威 + 承诺）
4. **我是否组合了太多原则？**（不要同时使用全部七种）
5. **这是否符合伦理？**（是否服务于用户的真实利益？）
